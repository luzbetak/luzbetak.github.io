Snowflake Experience and Modern Data Experience

Snowflake Experience

Snowflake hands-on skills with the Snowflake Data Cloud platform: building, operating, and optimizing a data warehouse in Snowflake.

Core Snowflake Skills

  • Data warehousing concepts in Snowflake
    • Designing schemas (star, snowflake, 3NF)
    • Understanding micro-partitions and pruning
    • Working with columnar storage and compression
  • Snowflake SQL
    • Complex joins and aggregations
    • Window functions
    • Using MERGE for upserts
    • Using COPY INTO for loading and unloading data
    • Multi-table INSERT patterns
  • Virtual warehouse management
    • Choosing appropriate warehouse sizes
    • Configuring auto-suspend and auto-resume
    • Scaling up vs. scaling out
    • Cost-control practices (separate warehouses by workload, etc.)
  • Data loading and unloading
    • Using internal stages
    • Using external stages (S3, Azure Blob, GCS)
    • Defining file formats (CSV, JSON, Parquet, ORC)
    • Bulk loads and incremental loads
  • Semi-structured data handling
    • Using the VARIANT data type
    • Storing and querying JSON, Parquet, Avro, XML
    • Using FLATTEN and LATERAL joins to normalize nested data
  • Performance optimization
    • Reading and interpreting query profiles
    • Choosing and managing clustering keys
    • Leveraging result cache, metadata cache, and data cache
    • Refactoring queries for better performance and lower cost
  • Security and governance in Snowflake
    • Role-based access control (RBAC) and role hierarchy
    • Grants on databases, schemas, tables, and views
    • Network policies and secure data sharing
    • Masking and row access policies
  • Advanced Snowflake features
    • Zero-copy cloning of databases, schemas, and tables
    • Time Travel for querying historical data
    • Fail-Safe for recovery
    • Streams & Tasks for change data capture and scheduling
    • Snowpipe for continuous data ingestion
    • Materialized views for performance
    • Snowpark (Python/Java/Scala) for data engineering in-code
    • Data sharing and Snowflake Marketplace

Modern Data Experience (Modern Data Stack)

Modern data experience (or modern data stack experience) usually means working with a cloud-native ecosystem of tools around storage, compute, transformation, orchestration, and BI, not just Snowflake itself.

Key Components of the Modern Data Stack

  1. Cloud data warehouses and lakehouses
    • Snowflake
    • Google BigQuery
    • Databricks SQL Warehouse
    • Amazon Redshift
  2. ETL / ELT and transformation tools
    • dbt / dbt Cloud
    • Airflow-based ETL / ELT
    • Dataform
    • Fivetran, Stitch, Matillion, Informatica Cloud
  3. Orchestration and workflow management
    • Apache Airflow (including MWAA / Cloud Composer)
    • Dagster
    • Prefect
    • AWS Step Functions
    • Azure Data Factory pipelines
  4. Data lake and storage layers
    • Amazon S3
    • Google Cloud Storage (GCS)
    • Azure Data Lake Storage (ADLS)
    • Table formats: Delta Lake, Apache Iceberg
  5. Data engineering computation frameworks
    • Apache Spark
    • PySpark
    • Databricks environment (Jobs, Notebooks, Workflows)
    • Snowpark for Snowflake-native compute
  6. BI and analytics tools
    • Looker
    • Tableau
    • Power BI
    • Mode Analytics
    • Sigma Computing
  7. Real-time and streaming technologies
    • Apache Kafka
    • Amazon Kinesis
    • Google Pub/Sub
    • Apache Flink
    • Streaming ETL tools like StreamSets
  8. DevOps / DataOps practices
    • CI/CD for SQL, dbt projects, and pipelines
    • Git-based workflows (branching, PRs, code review)
    • Infrastructure as Code (e.g., Terraform for Snowflake and cloud resources)
    • Monitoring and observability (Datadog, Monte Carlo, BigEye, etc.)
